# 🎨 Sprint 1 — Definición de Fundamentos UX/UI

## 🏢 Sector
EdTech

## 📅 Duración
Semana 1

---

## 🎯 Objetivo del Sprint
- Definir una arquitectura de marca clara y escalable  
- Crear una base reutilizable de componentes de diseño  
- Establecer lineamientos de experiencia de usuario consistentes  
- Incorporar criterios de accesibilidad desde etapas tempranas  
- Sentar las bases para una gobernanza sostenible del sistema  

---

## 🧩 Contexto del Proyecto
Luego del Sprint 0 (entendimiento del problema y alineación), el equipo avanza hacia la materialización del sistema, comenzando por:

- Identidad visual base  
- Principios UX  
- Estructura inicial del Design System  

---

## 📌 Alcance del Sprint 1

### 🎨 Definición de identidad visual base
Se establece el núcleo visual del sistema:

- Paleta de colores (primaria, secundaria, neutros, estados)  
- Tipografía (familias, jerarquías, escalas)  
- Espaciado y grid system  
- Estilos base (bordes, sombras, iconografía inicial)  

---

### 🧱 Fundamentos del Design System
Se define la estructura inicial del sistema:

- Design Tokens (color, spacing, typography)  
- Naming conventions  
- Organización en Figma (librerías, páginas, assets)  
- Primer esquema de versionado  

---

### 🧭 Principios de UX
Se establecen lineamientos que guiarán todas las decisiones:

- Consistencia  
- Claridad  
- Accesibilidad  
- Feedback inmediato  
- Eficiencia de uso  

---

### 🔍 Benchmark (continuación)
Se profundiza el análisis iniciado en Sprint 0:

- Análisis de Design Systems en EdTech  
- Evaluación de patrones UI comunes  
- Identificación de buenas prácticas y gaps  

---

### 🧩 Arquitectura inicial de componentes
Se comienza a estructurar:

- Clasificación de componentes (atoms, molecules, organisms)  
- Identificación de componentes core:
  - Botones  
  - Inputs  
  - Cards  
  - Navegación
